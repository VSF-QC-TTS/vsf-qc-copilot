# Story 06.02 - Create Dataset

> Epic: 6 - Datasets
> Depends on: 06.01, 05.01
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** to create a draft dataset under a project,
**so that** I can add or generate test cases before approval.

## In Scope

- [WEB] Create dataset dialog.
- [WEB] Optional requirement dropdown.
- [WEB] Send `sourceType: MANUAL` for manual UI creation.
- [WEB] Navigate to dataset detail after create.

## Out of Scope

- Creating test cases during dataset creation.

## Deferred

- Dataset templates.

## API/Data Contract

```text
POST /api/v1/projects/{projectPublicId}/datasets
Request: { requirementPublicId?, sourceType: 'MANUAL', name, description? }
Response: DatasetResponse
```

## UI States

- Requirement dropdown loading.
- Validation error for missing name.
- Submit loading.

## Acceptance Criteria

### AC 1 - Created dataset starts as draft

```text
Platform: [WEB]
Actor: QC member

Given  the member submits a valid dataset name
When   create succeeds
Then   the new dataset detail page opens
And    the dataset status is DRAFT
```

### AC 2 - Manual create sends source type

```text
Platform: [WEB]
Actor: QC member

Given  the member creates a dataset from the UI dialog
When   the request is sent
Then   sourceType is MANUAL
```

## Implementation Checklist

- Load active requirements for dropdown.
- Default manual creation to `sourceType: MANUAL`.
- Invalidate dataset list after create.

## Verification Notes

- Test create with and without requirementPublicId.
