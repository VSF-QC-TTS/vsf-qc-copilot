# Story 06.03 - Dataset Detail

> Epic: 6 - Datasets
> Depends on: 06.02, 07.01
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** to inspect and approve a dataset,
**so that** only ready test-case sets can be used for evaluation.

## In Scope

- [WEB] Dataset info panel.
- [WEB] Edit name/description.
- [WEB] Approve and archive actions.
- [WEB] Embed test case section from Epic 7.

## Out of Scope

- Reverting APPROVED to DRAFT.

## Deferred

- Dataset activity timeline.

## API/Data Contract

```text
GET /api/v1/datasets/{datasetPublicId}
PATCH /api/v1/datasets/{datasetPublicId}
```

Approval requires 1-100 active test cases.

## UI States

- DRAFT: editable, test cases mutable.
- APPROVED: test cases read-only for mutation flows.
- ARCHIVED: mutation actions hidden or disabled.

## Acceptance Criteria

### AC 1 - Approve validates active case count

```text
Platform: [WEB]
Actor: QC member

Given  a draft dataset has zero active test cases
When   the detail page renders
Then   the approve action is disabled with explanatory helper text
```

### AC 2 - Approved datasets lock test case edits

```text
Platform: [WEB]
Actor: QC member

Given  a dataset is APPROVED
When   the test case section renders
Then   create, edit, delete, import, and AI generate actions are unavailable
```

## Implementation Checklist

- Create dataset detail route.
- Add status transition confirmation dialogs.
- Render linked requirement summary when present.

## Verification Notes

- Test approval boundary at 1 and 100 active cases.
