# Story 12.02 - Error Boundaries

> Epic: 12 - Polish And Settings
> Depends on: all route groups
> Backend contract source: N/A

## Story Header

**As a** QC member,
**I want** clear route and API error states,
**so that** failures do not leave me in a blank or confusing screen.

## In Scope

- [WEB] Route-group error boundaries.
- [WEB] Locale-aware not-found page.
- [WEB] API error toast mapping.

## Out of Scope

- Error reporting service integration.

## Deferred

- Sentry or OpenTelemetry frontend reporting.

## API/Data Contract

- API errors expose `code`; frontend maps code to localized text.

## UI States

- Route error with retry.
- 404 not found.
- Network/server/API validation errors.
