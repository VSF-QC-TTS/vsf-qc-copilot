# Story 12.04 - Responsive Audit

> Epic: 12 - Polish And Settings
> Depends on: all feature pages
> Backend contract source: N/A

## Story Header

**As a** QC member using desktop, tablet, or mobile,
**I want** every screen to remain usable at supported viewport sizes,
**so that** urgent QC work is not blocked by layout failures.

## In Scope

- [WEB] Audit desktop 1280px+, tablet 768px, and mobile 375px.
- [WEB] Tables scroll inside their own container on mobile.
- [WEB] Dialogs and side panels adapt to full-screen mobile.
- [WEB] Touch targets at least 44px on mobile.

## Out of Scope

- Native mobile app behavior.

## Deferred

- Cross-browser matrix beyond the project-supported browsers.

## API/Data Contract

- No backend contract.

## UI States

- Long Vietnamese labels.
- Long project/dataset/question text.
- Empty/loading/error states at all breakpoints.

## Acceptance Criteria

### AC 1 - No page-level horizontal overflow

```text
Platform: [WEB]
Actor: QC member

Given  the viewport is 375px wide
When   each dashboard screen renders
Then   the page itself has no horizontal scrollbar
And    DataTable overflow is contained within the table region
```

### AC 2 - Panels become mobile dialogs

```text
Platform: [WEB]
Actor: QC member

Given  the viewport is mobile width
When   a result detail panel or editor opens
Then   it renders as a full-screen modal
```

## Implementation Checklist

- Test every route at 1280px, 768px, and 375px.
- Fix header breadcrumb compression.
- Verify chart responsiveness.

## Verification Notes

- Use realistic long text data during visual QA.
