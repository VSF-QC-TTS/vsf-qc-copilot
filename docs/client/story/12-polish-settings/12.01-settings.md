# Story 12.01 - Settings

> Epic: 12 - Polish And Settings
> Depends on: 02.03
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** to view my profile and preferences,
**so that** I can confirm account details and adjust language/theme choices.

## In Scope

- [WEB] Settings page.
- [WEB] Read-only profile from `/api/v1/users/me`.
- [WEB] Language and theme controls.

## Out of Scope

- Change password and profile update until backend endpoints exist.

## Deferred

- `PATCH /api/v1/users/me`.
- `PATCH /api/v1/users/me/password`.

## API/Data Contract

```text
GET /api/v1/users/me
```

## UI States

- Profile loading.
- Read-only profile display.
- Preference updates for language/theme.

## Acceptance Criteria

### AC 1 - Profile uses displayName

```text
Platform: [WEB]
Actor: QC member

Given  GET /api/v1/users/me returns displayName
When   settings renders
Then   displayName, email, role, status, and lastLoginAt are shown
```

### AC 2 - Unsupported account mutations are not shown

```text
Platform: [WEB]
Actor: QC member

Given  no password update endpoint exists
When   settings renders
Then   no change-password form is displayed
```

## Implementation Checklist

- Create settings page.
- Reuse LanguageSwitcher and ThemeToggle patterns.
- Show deferred account mutation note only if product wants visible copy; otherwise omit.

## Verification Notes

- Test settings with disabled/pending status if fixture exists.
