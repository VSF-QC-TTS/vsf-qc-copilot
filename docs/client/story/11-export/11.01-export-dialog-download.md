# Story 11.01 - Export Dialog And Download

> Epic: 11 - Export
> Depends on: 00.08, 09.03
> Backend contract source: `server/SERVER_CONTEXT.md`

## Story Header

**As a** QC member,
**I want** to export evaluation results to Excel or JSON,
**so that** I can share or archive QC evidence outside the dashboard.

## In Scope

- [WEB] Export dialog from run detail.
- [WEB] Excel and JSON format selection.
- [WEB] Async export job polling.
- [WEB] Download ready file.

## Out of Scope

- Custom column mapping.

## Deferred

- S3/R2 storage provider concerns; client only uses download endpoint.

## API/Data Contract

```text
POST /api/v1/evaluation-runs/{runPublicId}/exports
Request: { fileType: 'EXCEL' | 'JSON' }
Response: { exportPublicId, jobPublicId, status, message }

GET /api/v1/exports/{exportPublicId}
Response: { publicId, status, fileType, downloadUrl?, errorMessage?, ... }

GET /api/v1/exports/{exportPublicId}/file
```

## UI States

- Export queued.
- Export job running.
- Ready with download link.
- Failed with error message.

## Acceptance Criteria

### AC 1 - Create stores exportPublicId

```text
Platform: [WEB]
Actor: QC member

Given  the member starts an export
When   the backend returns exportPublicId and jobPublicId
Then   the client polls jobPublicId
And    later fetches GET /api/v1/exports/{exportPublicId}
```

### AC 2 - Ready export uses downloadUrl

```text
Platform: [WEB]
Actor: QC member

Given  export detail status is READY
When   the dialog renders the download action
Then   it uses downloadUrl or GET /api/v1/exports/{exportPublicId}/file
And    it does not use a legacy download field name
```

## Implementation Checklist

- Create export dialog.
- Use `useJobProgress`.
- After job completion, fetch export detail before download.

## Verification Notes

- Test pending, ready, and failed export metadata.
