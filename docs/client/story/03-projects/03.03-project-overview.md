# Story 03.03 - Project Overview

> Epic: 3 - Projects
> Depends on: 03.01
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** a project overview with supported actions and recent evaluations,
**so that** I can continue setup or inspect project activity.

## In Scope

- [WEB] Project detail page.
- [WEB] Project status and archive action.
- [WEB] Recent evaluation runs using project-scoped endpoint.
- [WEB] Quick links to connectors, requirements, datasets, rubrics, and evaluations.

## Out of Scope

- Unsupported aggregate stats that require new backend endpoints.

## Deferred

- Project score trend chart until enough run data and aggregation rules are available.

## API/Data Contract

```text
GET /api/v1/projects/{projectPublicId}
DELETE /api/v1/projects/{projectPublicId}
GET /api/v1/projects/{projectPublicId}/evaluation-runs?page=0&size=5&sort=createdAt,desc
```

## UI States

- Loading project detail.
- Archived project shows muted status and limited actions.
- Empty recent evaluations.

## Acceptance Criteria

### AC 1 - Project detail loads by public id

```text
Platform: [WEB]
Actor: QC member

Given  the route contains a project public id
When   the page loads
Then   GET /api/v1/projects/{projectPublicId} is called
And    the project name and status are shown
```

### AC 2 - Archive is confirmed

```text
Platform: [WEB]
Actor: QC member

Given  the member chooses archive
When   the confirmation dialog is accepted
Then   DELETE /api/v1/projects/{projectPublicId} is called
```

## Implementation Checklist

- Create project overview page.
- Use supported recent evaluation endpoint only.
- Avoid fake metric cards for unavailable aggregates.
