# Story 02.04 - Dashboard Home

> Epic: 2 - Dashboard Shell
> Depends on: 02.03, 03.01 when project data is available
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** a simple dashboard landing page,
**so that** I can start from projects without needing unavailable aggregate APIs.

## In Scope

- [WEB] Welcome header using `user.displayName`.
- [WEB] Project count from `/api/v1/projects`.
- [WEB] Quick links to create/view projects.
- [WEB] Empty state for new users.

## Out of Scope

- Cross-project recent runs.
- Cross-project running jobs.
- Global average score.

## Deferred

- Dashboard aggregate endpoint.
- Cross-project recent evaluations endpoint.

## API/Data Contract

```text
GET /api/v1/users/me
GET /api/v1/projects?page=0&size=100
```

## UI States

- Loading: welcome and metrics skeleton.
- Empty: create first project CTA.
- Limited-data mode: show only metrics supported by current endpoints.

## Acceptance Criteria

### AC 1 - Welcome uses backend user shape

```text
Platform: [WEB]
Actor: QC member

Given  GET /api/v1/users/me returns displayName
When   dashboard home renders
Then   the welcome header greets the member by displayName
```

### AC 2 - Unsupported widgets are absent

```text
Platform: [WEB]
Actor: QC member

Given  no aggregate backend endpoint exists
When   dashboard home renders
Then   it does not fabricate global average score or cross-project recent runs
```
