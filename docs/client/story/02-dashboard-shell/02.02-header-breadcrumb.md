# Story 02.02 - Header And Breadcrumb

> Epic: 2 - Dashboard Shell
> Depends on: 02.01
> Backend contract source: feature detail endpoints as needed

## Story Header

**As a** QC member,
**I want** a header with breadcrumbs and account controls,
**so that** I always know where I am and can manage session preferences.

## In Scope

- [WEB] Sticky 64px header.
- [WEB] Breadcrumb based on route and loaded entity names.
- [WEB] Language switcher, theme toggle, user menu.

## Out of Scope

- Global search.

## Deferred

- Breadcrumb prefetch optimization.

## API/Data Contract

- Entity names may require current page detail data already loaded by the screen.

## UI States

- Breadcrumb collapses on mobile.
- User menu shows `displayName`, email, role.

## Acceptance Criteria

### AC 1 - Breadcrumb reflects route

```text
Platform: [WEB]
Actor: QC member

Given  the member views a dataset detail page
When   the header renders
Then   breadcrumb segments show Projects, project name, Datasets, and dataset name when data is available
```

## Implementation Checklist

- Create `Header`, `Breadcrumb`, and `UserMenu`.
- Use i18n labels for static route segments.
- Logout calls auth API and clears auth store.

## Verification Notes

- Test with long project and dataset names.
