# Story 08.03 - Rubric Detail And Versions

> Epic: 8 - Rubrics
> Depends on: 08.02
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** to manage rubric metadata and versions,
**so that** I can publish stable criteria sets for evaluations.

## In Scope

- [WEB] Rubric detail page.
- [WEB] Edit metadata and archive rubric.
- [WEB] Version list.
- [WEB] Create draft version, publish, and archive version.

## Out of Scope

- Criteria editing; covered by 08.05.

## Deferred

- Version comparison.

## API/Data Contract

```text
GET /api/v1/rubrics/{rubricPublicId}
PATCH /api/v1/rubrics/{rubricPublicId}
DELETE /api/v1/rubrics/{rubricPublicId}
GET /api/v1/rubrics/{rubricPublicId}/versions
POST /api/v1/rubrics/{rubricPublicId}/versions
PATCH /api/v1/rubric-versions/{rubricVersionPublicId}
```

Publishing requires at least one criterion.

## UI States

- No current version.
- Draft version.
- Published/archived versions read-only for criteria mutation.

## Acceptance Criteria

### AC 1 - New version is draft

```text
Platform: [WEB]
Actor: QC member

Given  the member clicks create new version
When   POST /api/v1/rubrics/{rubricPublicId}/versions succeeds
Then   a DRAFT version appears in the version list
```

### AC 2 - Publish failure is visible

```text
Platform: [WEB]
Actor: QC member

Given  a draft version has no criteria
When   the member attempts publish
Then   the backend validation error is shown in context
```

## Implementation Checklist

- Create rubric detail route.
- Add version table with actions.
- Use ConfirmDialog for archive and publish.

## Verification Notes

- Verify archived rubrics hide mutation actions.
