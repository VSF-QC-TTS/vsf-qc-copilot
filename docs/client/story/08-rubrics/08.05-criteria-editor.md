# Story 08.05 - Criteria Editor

> Epic: 8 - Rubrics
> Depends on: 08.04
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** to create and edit rubric criteria,
**so that** the AI judge receives clear instructions and scoring weights.

## In Scope

- [WEB] Criteria create/edit panel or dialog.
- [WEB] Fields: name, description, weight, judgeInstruction, passCondition,
  failCondition, metricKey, isCritical, sortOrder.
- [WEB] Delete criterion with confirmation.

## Out of Scope

- Editing criteria in non-DRAFT versions.

## Deferred

- AI-assisted criterion writing.

## API/Data Contract

```text
POST /api/v1/rubric-versions/{rubricVersionPublicId}/criteria
PATCH /api/v1/rubric-criteria/{criterionPublicId}
DELETE /api/v1/rubric-criteria/{criterionPublicId}
```

`metricKey` must be lowercase letters, numbers, and underscores, and unique within
the version.

## UI States

- Create mode.
- Edit mode.
- Non-DRAFT read-only mode.
- Delete confirmation.

## Acceptance Criteria

### AC 1 - Metric key validation is local and backend-backed

```text
Platform: [WEB]
Actor: QC member

Given  the member enters an invalid metricKey
When   they submit the form
Then   client validation blocks the request
```

### AC 2 - Critical criterion toggle persists

```text
Platform: [WEB]
Actor: QC member

Given  the member enables isCritical
When   the criterion is saved
Then   the saved criterion displays the critical badge in the list
```

## Implementation Checklist

- Add weight number input and slider synced to the same value.
- Disable all fields when version status is not DRAFT.
- Refresh criteria list after mutations.
