# Story 08.04 - Version Detail Criteria List

> Epic: 8 - Rubrics
> Depends on: 08.03
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** to inspect criteria inside a rubric version,
**so that** I can understand how evaluation scores will be computed.

## In Scope

- [WEB] Version detail page.
- [WEB] Criteria list sorted by `sortOrder`.
- [WEB] Relative weight display.
- [WEB] Add/edit entry point only for DRAFT versions.

## Out of Scope

- Drag-and-drop reorder unless backend supports reorder through criterion update.

## Deferred

- Interactive weight distribution editing.

## API/Data Contract

```text
GET /api/v1/rubric-versions/{rubricVersionPublicId}
GET /api/v1/rubric-versions/{rubricVersionPublicId}/criteria
```

Criteria weights are relative integers 1-100; evaluation normalizes them.

## UI States

- Empty draft criteria list.
- Read-only published/archived version.

## Acceptance Criteria

### AC 1 - Relative weights are derived client-side

```text
Platform: [WEB]
Actor: QC member

Given  criteria weights are 2 and 3
When   the criteria list renders
Then   relative weights are shown as 40% and 60%
```
