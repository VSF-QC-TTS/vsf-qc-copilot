# Story 07.04 - AI Generate

> Epic: 7 - Test Cases
> Depends on: 00.08, 05.01, 07.01
> Backend contract source: `server/SERVER_CONTEXT.md`

## Story Header

**As a** QC member,
**I want** to generate test cases from a requirement,
**so that** I can accelerate dataset creation while staying within dataset limits.

## In Scope

- [WEB] Generate dialog in draft datasets.
- [WEB] Requirement dropdown, count, additional prompt.
- [WEB] Submit async generation job.
- [WEB] Poll job progress with `useJobProgress`.

## Out of Scope

- SSE stream.
- Editing AI output before persistence.

## Deferred

- Prompt preview and regenerate selected cases.

## API/Data Contract

```text
POST /api/v1/datasets/{datasetPublicId}/generate
Request: { requirementPublicId, count, additionalPrompt? }
Response: { datasetPublicId, jobPublicId, status, message }

GET /api/v1/jobs/{jobPublicId}
```

`count` is 5-100 and cannot cause the dataset to exceed 100 active test cases.

## UI States

- Requirements loading.
- Job pending/running/completed/failed.
- Limit exceeded error.

## Acceptance Criteria

### AC 1 - Count enforces backend range

```text
Platform: [WEB]
Actor: QC member

Given  the generate dialog is open
When   the member enters a count below 5 or above 100
Then   the form blocks submit with a localized validation error
```

### AC 2 - Completed job refreshes cases

```text
Platform: [WEB]
Actor: QC member

Given  generation job polling returns COMPLETED
When   the hook handles completion
Then   the test case table is invalidated and refreshed
```

## Implementation Checklist

- Use polling-first `JobProgress`.
- Disable submit for non-DRAFT datasets.
- Show active case limit helper text.

## Verification Notes

- Test existing count + requested count over 100.
