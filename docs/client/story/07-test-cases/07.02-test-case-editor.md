# Story 07.02 - Test Case Editor

> Epic: 7 - Test Cases
> Depends on: 07.01
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** to create, inspect, edit, and delete individual test cases,
**so that** dataset content stays accurate before approval.

## In Scope

- [WEB] Side panel or modal editor.
- [WEB] Question, groundTruth, precondition, metadata fields.
- [WEB] Hard delete with confirmation.

## Out of Scope

- Editing test cases in APPROVED or ARCHIVED datasets.

## Deferred

- Visual diff for edited test cases.

## API/Data Contract

```text
POST /api/v1/datasets/{datasetPublicId}/test-cases
PATCH /api/v1/test-cases/{testCasePublicId}
DELETE /api/v1/test-cases/{testCasePublicId}
```

## UI States

- Create mode.
- View mode.
- Edit mode.
- Delete confirmation.

## Acceptance Criteria

### AC 1 - Metadata validates as JSON

```text
Platform: [WEB]
Actor: QC member

Given  the member enters metadata text
When   the metadata is not valid JSON
Then   the form blocks submit and shows an inline validation error
```

### AC 2 - Delete refreshes table

```text
Platform: [WEB]
Actor: QC member

Given  a test case is open in the editor
When   the member confirms delete
Then   DELETE /api/v1/test-cases/{testCasePublicId} is called
And    the table refreshes after success
```

## Implementation Checklist

- Reuse FormField and ConfirmDialog.
- Close panel after successful save/delete.
- Keep panel full-screen on mobile.

## Verification Notes

- Test parent dataset status restrictions.
