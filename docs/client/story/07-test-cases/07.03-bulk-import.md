# Story 07.03 - Bulk Import

> Epic: 7 - Test Cases
> Depends on: 07.01
> Backend contract source: `server/src/main/java/.../TestCaseImportServiceImpl.java`

## Story Header

**As a** QC member,
**I want** to import test cases from CSV or Excel,
**so that** I can prepare datasets faster than entering each case manually.

## In Scope

- [WEB] File upload dialog.
- [WEB] `.xlsx` and `.csv` only.
- [WEB] Max file size 5 MB.
- [WEB] Import result summary and row errors.

## Out of Scope

- Importing more than the dataset 100 active case limit.

## Deferred

- Template download.

## API/Data Contract

```text
POST /api/v1/datasets/{datasetPublicId}/test-cases/import
Content-Type: multipart/form-data
Body: file
Response: { totalRows, importedCount, skippedCount, errors: [{ row, field, message }] }
```

Backend rejects import when existing active cases plus file rows exceed 100.

## UI States

- Drag over.
- Uploading.
- Success with imported/skipped counts.
- Invalid file type, file too large, too many rows.
