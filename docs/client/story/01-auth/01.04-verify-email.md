# Story 01.04 - Verify Email

> Epic: 1 - Authentication
> Depends on: 01.01
> Backend contract source: `server/SERVER_CONTEXT.md`

## Story Header

**As a** newly registered QC member,
**I want** the verification link to activate my account,
**so that** I can log in after confirming my email.

## In Scope

- [WEB] Read `token` from query string.
- [WEB] Auto-submit verification request once.
- [WEB] Loading, success, and failure states.

## Out of Scope

- Requesting a new verification email.

## Deferred

- Resend verification email endpoint.

## API/Data Contract

```text
POST /api/v1/auth/verify-email
Request: { token }
Response: UserResponse
```

## UI States

- Missing token: show invalid link error.
- Loading: verification in progress.
- Success: link to login.
- Error: expired or invalid token message.

## Acceptance Criteria

### AC 1 - Token verifies automatically

```text
Platform: [WEB]
Actor: Newly registered QC member

Given  the URL contains a verification token
When   the page mounts
Then   POST /api/v1/auth/verify-email is called once
```

## Implementation Checklist

- Create verify-email page.
- Use search params for token.
- Prevent duplicate submission on rerender.

## Verification Notes
