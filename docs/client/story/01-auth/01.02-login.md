# Story 01.02 - Login

> Epic: 1 - Authentication
> Depends on: 00.02, 00.03, 01.01
> Backend contract source: `server/SERVER_CONTEXT.md`

## Story Header

**As a** registered QC member,
**I want** to log in with email and password,
**so that** I can access protected dashboard workflows.

## In Scope

- [WEB] Email/password login form.
- [WEB] React Hook Form + Zod validation.
- [WEB] Store access token and user in memory on success.

## Out of Scope

- Remember-me token persistence.
- OAuth login.

## Deferred

- Resend verification email action unless backend endpoint is added.

## API/Data Contract

```text
POST /api/v1/auth/login
Request: { email, password }
Response: { accessToken, tokenType, expiresInSeconds, user }
```

## UI States

- Loading submit button.
- Inline invalid credential or unverified email error.
- Network errors use toast.

## Acceptance Criteria

### AC 1 - Valid login redirects

```text
Platform: [WEB]
Actor: Registered QC member

Given  the member enters valid credentials
When   login succeeds
Then   the access token and user are stored in memory
And    the member is redirected to /vi/dashboard
```

### AC 2 - Invalid login stays on form

```text
Platform: [WEB]
Actor: Registered QC member

Given  the member enters invalid credentials
When   the backend returns an auth error
Then   the login page shows an inline localized error
```

## Implementation Checklist

- Create `app/[locale]/(auth)/login/page.tsx`.
- Add `lib/api/auth.ts` login function.
- Add `lib/validations/auth.ts` login schema.
- Link to register and forgot-password routes.

## Verification Notes

- Verify no token is written to localStorage.
