# Story 01.05 - Forgot And Reset Password

> Epic: 1 - Authentication
> Depends on: 01.01
> Backend contract source: `server/SERVER_CONTEXT.md`

## Story Header

**As a** QC member who forgot my password,
**I want** to request a reset link and set a new password,
**so that** I can recover account access without exposing account existence.

## In Scope

- [WEB] Forgot-password email form.
- [WEB] Reset-password token form.
- [WEB] Always-success forgot-password UX.

## Out of Scope

- Changing password while already logged in.

## Deferred

- Settings password update until backend endpoint exists.

## API/Data Contract

```text
POST /api/v1/auth/forgot-password
Request: { email }
Response: 204

POST /api/v1/auth/reset-password
Request: { token, newPassword }
```

## UI States

- Forgot-password always shows the same success message after submit.
- Reset page handles missing, invalid, and expired token errors.

## Acceptance Criteria

### AC 1 - Forgot password does not reveal accounts

```text
Platform: [WEB]
Actor: QC member

Given  the member submits any syntactically valid email
When   forgot-password returns
Then   the page shows the same success message regardless of account existence
```

### AC 2 - Reset password validates confirmation

```text
Platform: [WEB]
Actor: QC member

Given  the reset page has a token
When   the member enters mismatched passwords
Then   the form blocks submit and shows an inline error
```

## Implementation Checklist

- Create forgot-password and reset-password pages.
- Reuse password schema logic from register.
- Link back to login.

## Verification Notes

- Confirm no account-existence message appears.
