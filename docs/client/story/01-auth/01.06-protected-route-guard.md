# Story 01.06 - Protected Route Guard

> Epic: 1 - Authentication
> Depends on: 00.02, 00.03, 00.07
> Backend contract source: `server/SERVER_CONTEXT.md`

## Story Header

**As a** QC member,
**I want** protected dashboard pages to restore or reject my session correctly,
**so that** reloads do not incorrectly trap me on login or expose private pages.

## In Scope

- [WEB] Locale middleware only handles locale redirects.
- [WEB] Client-side dashboard guard bootstraps auth by attempting refresh.
- [WEB] Redirect unauthenticated users to locale-aware login.

## Out of Scope

- Middleware reading Zustand state.
- Server-side protected page rendering.

## Deferred

- BFF/same-origin session cookie model.

## API/Data Contract

```text
POST /api/v1/auth/refresh-token
GET /api/v1/users/me
```

## UI States

- Bootstrapping: dashboard shell may show a skeleton.
- Authenticated: render protected content.
- Unauthenticated: redirect to `/{locale}/login`.

## Acceptance Criteria

### AC 1 - Reload attempts refresh

```text
Platform: [WEB]
Actor: Previously authenticated QC member

Given  the browser reloads a protected dashboard route
When   the in-memory token is empty
Then   the auth guard attempts POST /api/v1/auth/refresh-token before redirecting
```

### AC 2 - Failed refresh redirects

```text
Platform: [WEB]
Actor: Unauthenticated visitor

Given  refresh-token fails
When   the visitor is on /vi/projects
Then   the app redirects to /vi/login
```

## Implementation Checklist

- Combine `next-intl` middleware for locale routing only.
- Add dashboard auth bootstrap provider/guard.
- Avoid relying on middleware to read JS memory state.

## Verification Notes

- Test direct navigation to `/vi/dashboard` after reload.
