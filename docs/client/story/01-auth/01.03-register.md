# Story 01.03 - Register

> Epic: 1 - Authentication
> Depends on: 01.01
> Backend contract source: `server/SERVER_CONTEXT.md`

## Story Header

**As a** new QC member,
**I want** to register with email, password, and display name,
**so that** I can create an account pending email verification.

## In Scope

- [WEB] Registration form.
- [WEB] Password confirmation and strength indicator.
- [WEB] Success state instructing the user to check email.

## Out of Scope

- Auto-login after registration.

## Deferred

- Admin invite flow.

## API/Data Contract

```text
POST /api/v1/auth/register
Request: { email, password, fullName }
Response: UserResponse
```

Backend persists pending users and sends a verification email.

## UI States

- Validation errors below fields.
- Duplicate email error.
- Success message replaces the form.

## Acceptance Criteria

### AC 1 - Registration submits valid payload

```text
Platform: [WEB]
Actor: New QC member

Given  the member enters a valid email, password, confirmation, and name
When   they submit the form
Then   POST /api/v1/auth/register is called with email, password, and fullName
```

### AC 2 - Success asks user to verify email

```text
Platform: [WEB]
Actor: New QC member

Given  registration succeeds
When   the page updates
Then   the user sees a localized check-email message
```

## Implementation Checklist

- Add register page and schema.
- Validate password minimum and confirmation match.
- Link back to login.

## Verification Notes

- Confirm returned role displays as `QC_MEMBER` if shown.
