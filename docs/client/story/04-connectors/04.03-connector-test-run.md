# Story 04.03 - Connector Test Run

> Epic: 4 - Target API Connectors
> Depends on: 04.02
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** to test a connector with a sample question,
**so that** I can verify request templates and response extraction before running evaluations.

## In Scope

- [WEB] Test-run panel on connector detail.
- [WEB] Question, precondition, and metadata inputs.
- [WEB] Response status, latency, body, extracted answer, and masked preview headers.

## Out of Scope

- Saving test-run history.

## Deferred

- Advanced response selector debugger.

## API/Data Contract

```text
POST /api/v1/target-api-connectors/{connectorPublicId}/test-runs
```

Backend resolves connector secrets at runtime and never returns raw secret values.

## UI States

- Loading while request runs.
- Success response summary.
- Timeout/connection/error response.

## Acceptance Criteria

### AC 1 - Test run submits sample data

```text
Platform: [WEB]
Actor: QC member

Given  a connector detail page is open
When   the member enters a sample question and runs the test
Then   POST /api/v1/target-api-connectors/{connectorPublicId}/test-runs is called
```

### AC 2 - Secrets are not exposed

```text
Platform: [WEB]
Actor: QC member

Given  the connector uses secret placeholders
When   the test-run result is displayed
Then   raw secret values are not visible anywhere in the UI
```

## Implementation Checklist

- Add expandable test panel.
- Render JSON response in a readable code block.
- Highlight extracted answer separately.

## Verification Notes

- Test 2xx, 4xx, timeout, and malformed response cases.
