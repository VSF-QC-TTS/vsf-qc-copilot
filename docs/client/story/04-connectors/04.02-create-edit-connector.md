# Story 04.02 - Create/Edit Connector

> Epic: 4 - Target API Connectors
> Depends on: 04.01
> Backend contract source: `server/src/main/java/.../CreateTargetApiConnectorRequest.java`

## Story Header

**As a** QC member,
**I want** to configure a target API connector with templates and secrets,
**so that** evaluations can call the chatbot endpoint safely.

## In Scope

- [WEB] Create and edit connector form.
- [WEB] Headers, query params, path params, auth config, and body templates.
- [WEB] Write-only `secretValues`.
- [WEB] `responseSelector`, timeout, retry count, and active flag.

## Out of Scope

- Raw cURL parser automation unless already available.
- Streaming target API support beyond preserving fields.

## Deferred

- Rich JSON editor can be replaced by textarea for MVP.

## API/Data Contract

```text
POST /api/v1/projects/{projectPublicId}/target-api-connectors
PATCH /api/v1/target-api-connectors/{connectorPublicId}
Request fields include:
name, description, rawCurl, protocol, method, baseUrl, path, url,
headers, queryParams, pathParams, bodyType, bodyTemplate, bodyTemplateText,
authType, authConfig, secretValues, responseFormat, responseSelector,
isStreaming, streamingType, streamingEventSelector, timeoutSeconds, retryCount, active
```

## UI States

- Saved secrets show only masked references from `secretRefs`.
- Invalid URL or missing response selector blocks submit.
- Archived/inactive connector can still be viewed but active flag controls use.

## Acceptance Criteria

### AC 1 - Create uses current DTO names

```text
Platform: [WEB]
Actor: QC member

Given  the member submits a connector form
When   the request is sent
Then   the payload uses responseSelector and secretValues
And    it does not send legacy selector or secret field names
```

### AC 2 - Raw secrets are write-only

```text
Platform: [WEB]
Actor: QC member

Given  an existing connector is loaded
When   the form displays secret fields
Then   raw secret values are not shown
And    masked secretRefs are shown instead
```

## Implementation Checklist

- Create shared key-value editor.
- Support JSON object parsing for map fields.
- Validate timeout 1-300 and retry count 0-5.
- Keep helper text for `{{question}}`, `{{precondition}}`, and `{{metadata}}`.
