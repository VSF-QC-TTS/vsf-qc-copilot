# Story 05.01 - Requirement CRUD

> Epic: 5 - Requirements
> Depends on: 03.03
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** to manage project requirements,
**so that** datasets and AI-generated test cases can reference the intended chatbot behavior.

## In Scope

- [WEB] Project-scoped requirement list.
- [WEB] Inline create and edit.
- [WEB] Archive by PATCH status.
- [WEB] Version number display.

## Out of Scope

- Requirement diff view.

## Deferred

- Rich text requirement editor.

## API/Data Contract

```text
GET /api/v1/projects/{projectPublicId}/requirements?page=0&size=50&status=ACTIVE
POST /api/v1/projects/{projectPublicId}/requirements
PATCH /api/v1/requirements/{requirementPublicId}
```

`PATCH` increments `version` only when content changes.

## UI States

- Empty requirement state.
- Expanded full-content view.
- Archived status filter.

## Acceptance Criteria

### AC 1 - Create trims content through backend

```text
Platform: [WEB]
Actor: QC member

Given  the member enters non-empty requirement content
When   they save
Then   POST /api/v1/projects/{projectPublicId}/requirements is called
And    the new requirement appears in the list
```

### AC 2 - Archive uses PATCH

```text
Platform: [WEB]
Actor: QC member

Given  an active requirement is visible
When   the member confirms archive
Then   PATCH /api/v1/requirements/{requirementPublicId} is called with status ARCHIVED
```

## Implementation Checklist

- Build a simple card/list view rather than DataTable.
- Add active/archived filters.
- Show version number and updated date.
