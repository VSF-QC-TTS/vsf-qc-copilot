# Story 09.02 - Start Evaluation

> Epic: 9 - Evaluation Runs
> Depends on: 00.08, 04.03, 06.03, 08.05
> Backend contract source: `server/SERVER_CONTEXT.md`

## Story Header

**As a** QC member,
**I want** to start an evaluation with an approved dataset, published rubric version, and connector,
**so that** the backend can judge chatbot answers asynchronously.

## In Scope

- [WEB] Start evaluation dialog.
- [WEB] Dataset, rubric version, and connector selectors.
- [WEB] Quick-evaluate mode.
- [WEB] Poll returned job until terminal.

## Out of Scope

- SSE stream.
- Running promptfoo in the browser.

## Deferred

- Auto-select UX can remain simple until dashboard analytics exist.

## API/Data Contract

```text
POST /api/v1/projects/{projectPublicId}/evaluation-runs
Request: { datasetPublicId, rubricVersionPublicId, connectorPublicId }
Response: { runPublicId, jobPublicId, status, message }

POST /api/v1/projects/{projectPublicId}/quick-evaluate
Request: { datasetPublicId?, rubricVersionPublicId?, connectorPublicId? }
Response: { runPublicId, jobPublicId, status, message }
```

Quick evaluate allows null fields and backend resolves only when exactly one
candidate exists.

## UI States

- Selector loading.
- Business validation error for zero/multiple quick-evaluate candidates.
- Pending/running job progress using polling.

## Acceptance Criteria

```text

```
