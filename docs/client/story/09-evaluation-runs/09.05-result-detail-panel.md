# Story 09.05 - Result Detail Panel

> Epic: 9 - Evaluation Runs
> Depends on: 09.04, 10.01
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** a side panel with full result context,
**so that** I can review question, expected answer, actual answer, judge details, and QC decision in one place.

## In Scope

- [WEB] Slide-in result detail panel.
- [WEB] Question, precondition, ground truth, actual answer.
- [WEB] Judge status, score, and per-criterion JSON breakdown.
- [WEB] Embedded QC review form.
- [WEB] Prev/Next navigation.

## Out of Scope

- Separate result detail route.

## Deferred

- Visual diff view.

## API/Data Contract

- Uses selected row data from results list.
- Fetches review decision through Epic 10 if needed:
  `GET /api/v1/evaluation-results/{resultPublicId}/review-decision`.

## UI States

- Open/closed.
- Mobile full-screen modal.
- Missing optional fields render empty labels, not errors.

## Acceptance Criteria

### AC 1 - Panel keeps route context

```text
Platform: [WEB]
Actor: QC member

Given  the member is on the results page
When   they open a result panel
Then   the browser remains on the results route
```

### AC 2 - Keyboard close works

```text
Platform: [WEB]
Actor: QC member

Given  the panel is open
When   the member presses Escape
Then   the panel closes and focus returns to the triggering row
```
