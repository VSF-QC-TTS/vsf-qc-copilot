# Story 09.04 - Results Table

> Epic: 9 - Evaluation Runs
> Depends on: 09.03
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** to filter and open evaluation results,
**so that** I can focus review work on failed, warning, or unreviewed cases.

## In Scope

- [WEB] Results DataTable.
- [WEB] Judge status and QC status filters.
- [WEB] Row click opens result detail panel.

## Out of Scope

- Bulk review action unless backend batch endpoint exists.

## Deferred

- Bulk mark as reviewed.

## API/Data Contract

```text
GET /api/v1/evaluation-runs/{runPublicId}/results?page=0&size=20&judgeStatus=PASS&qcStatus=NOT_REVIEWED
```

Response includes test-case review context, QC fields, and structured `criteriaResults`
for compact pass/fail/error summaries.

## UI States

- Empty result state.
- NOT_REVIEWED indicator.
- Filtered no-result state.

## Acceptance Criteria

### AC 1 - Filters map to backend params

```text
Platform: [WEB]
Actor: QC member

Given  the member selects judgeStatus FAIL
When   the table refetches
Then   the query includes judgeStatus=FAIL
```

### AC 2 - Row opens panel

```text
Platform: [WEB]
Actor: QC member

Given  results are visible
When   the member clicks a row
Then   the result detail panel opens without leaving the results route
```

## Implementation Checklist

- Create results page.
- Keep selected result in URL search param or local state.
- Invalidate row after review mutations.
- Prefer `criteriaResults` and keep `criteriaResultsJson` as fallback for older rows.
