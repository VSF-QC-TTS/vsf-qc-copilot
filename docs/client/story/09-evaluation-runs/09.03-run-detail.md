# Story 09.03 - Run Detail

> Epic: 9 - Evaluation Runs
> Depends on: 09.02
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** to inspect an evaluation run summary and events,
**so that** I can understand job status and high-level results before reviewing cases.

## In Scope

- [WEB] Run detail page.
- [WEB] Summary cards from run detail fields.
- [WEB] Job progress polling when status is active.
- [WEB] Job events timeline.
- [WEB] Link to detailed results.

## Out of Scope

- Per-criterion aggregate chart if backend does not provide aggregate data.

## Deferred

- Score trend across multiple runs.

## API/Data Contract

```text
GET /api/v1/evaluation-runs/{runPublicId}
GET /api/v1/evaluation-runs/{runPublicId}/events
GET /api/v1/jobs/{jobPublicId}
```

`JobDetailResponse` contains `progressCurrent` and `progressTotal`; percent is a
client-derived display value.

## UI States

- Loading run detail.
- Running progress.
- Failed run with error event/message.
- Completed run with result link and export action.

## Acceptance Criteria

### AC 1 - Events render chronologically

```text
Platform: [WEB]
Actor: QC member

Given  run events are returned by the backend
When   the timeline renders
Then   events are shown in chronological order
```

### AC 2 - Progress percent is derived safely

```text
Platform: [WEB]
Actor: QC member

Given  a job has progressCurrent and progressTotal
When   JobProgress renders
Then   percent is calculated only when progressTotal is greater than zero
```
