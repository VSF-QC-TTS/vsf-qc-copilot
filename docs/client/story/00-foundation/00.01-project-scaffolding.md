# Story 00.01 - Project Scaffolding

> Epic: 0 - Foundation
> Depends on: none
> Backend contract source: N/A

## Story Header

**As a** frontend implementation agent,
**I want** a configured Next.js dashboard project,
**so that** all later screens can be built on the same tooling and design tokens.

## In Scope

- [WEB] Create the `client/` Next.js App Router project.
- [WEB] Configure Tailwind CSS v4, shadcn/ui, TypeScript, linting, and base env.
- [WEB] Add Geist Sans and Geist Mono through `next/font`.

## Out of Scope

- Feature routes.
- API client behavior.
- Authentication logic.

## Deferred

- None.

## API/Data Contract

- `.env.local` defines `NEXT_PUBLIC_API_URL=http://localhost:8080`.

## UI States

- Initial root page can be a minimal placeholder.
- Visual validation only requires theme tokens and a shadcn `Button`.

## Acceptance Criteria

### AC 1 - Project boots

```text
Platform: [WEB]
Actor: Frontend implementation agent

Given  the client project has been scaffolded
When   the agent runs the local dev command
Then   the Next.js app starts without compile errors
```

### AC 2 - Design tokens exist

```text
Platform: [WEB]
Actor: Frontend implementation agent

Given  the root layout renders
When   the app loads in the browser
Then   Geist fonts and light/dark CSS variables are available
```

## Implementation Checklist

- Create `client/package.json`, `next.config.ts`, `tsconfig.json`, and style config.
- Initialize shadcn/ui and customize tokens to zinc neutrals plus electric blue.
- Add `src/app/layout.tsx`, `src/app/[locale]/layout.tsx`, and `src/styles/globals.css`.
- Add `.env.local` with local API base URL.

## Verification Notes

- Run `rtk pnpm run build` from `client/` when dependencies are installed.
