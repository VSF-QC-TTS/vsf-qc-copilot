# Story 00.02 - API Client Layer

> Epic: 0 - Foundation
> Depends on: 00.01
> Backend contract source: `server/SERVER_CONTEXT.md`, `server/API_TREE.md`

## Story Header

**As a** frontend implementation agent,
**I want** one typed API client and DTO source,
**so that** screens do not duplicate fetch, auth refresh, pagination, or error handling.

## In Scope

- [WEB] Create a typed fetch wrapper in `src/lib/api/client.ts`.
- [WEB] Create backend DTO types in `src/lib/api/types.ts`.
- [WEB] Configure TanStack Query providers.

## Out of Scope

- Endpoint-specific hooks beyond examples.
- SSE streaming.

## Deferred

- Generated OpenAPI client is deferred until backend publishes a stable spec.

## API/Data Contract

```ts
type PageResponse<T> = {
  items: T[];
  page: number;
  size: number;
  totalItems: number;
  totalPages: number;
};
```

Errors are Problem Details-like payloads with at least `status`, `code`, and
human-readable detail/message when provided by the backend.

## UI States

- Query loading and error presentation are owned by calling screens.
- Global query errors may show toast only for transient failures.

## Acceptance Criteria

### AC 1 - Typed GET works

```text
Platform: [WEB]
Actor: Frontend implementation agent

Given  the user has a valid access token in memory
When   the app calls GET /api/v1/users/me through apiClient
Then   the response is typed as UserResponse
```

### AC 2 - Refresh retry works

```text
Platform: [WEB]
Actor: Authenticated QC member

Given  the access token has expired
When   a protected request returns 401
Then   apiClient calls POST /api/v1/auth/refresh-token
And    retries the original request once after receiving a new access token
```

### AC 3 - Refresh failure clears auth

```text
Platform: [WEB]
Actor: Authenticated QC member

Given  the access token has expired
When   refresh-token also fails
Then   the auth store is cleared
And    the user is sent to the locale-aware login page
```

## Implementation Checklist

- Use `NEXT_PUBLIC_API_URL` as base URL.
- Attach `Authorization: Bearer {accessToken}` only when a token exists.
- Implement `get`, `post`, `patch`, `put`, and `del`.
- Normalize API errors into a single client error type.
- Register `QueryClientProvider` in the app provider tree.

## Verification Notes

- Search implementation code for old pagination property names before merging.
