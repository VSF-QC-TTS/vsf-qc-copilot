# Story 00.08 - Job Progress Polling / Event Streaming

> Epic: 0 - Foundation
> Depends on: 00.02
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** async jobs to show progress using existing backend endpoints and evaluation event streams,
**so that** I can track evaluation, export, and AI generation without waiting for manual refresh.

## In Scope

- [WEB] Poll `GET /api/v1/jobs/{jobPublicId}` every 3 seconds while a job is active.
- [WEB] Expose a `useJobProgress` hook.
- [WEB] Render shared `JobProgress` UI.
- [WEB] For evaluation run details, stream `GET /api/v1/evaluation-runs/{runPublicId}/events/stream` with fetch streaming and Bearer auth.

## Out of Scope

- Native `EventSource` with token query params.
- Cancel job action unless backend adds an endpoint.

## Deferred

- Generic `GET /api/v1/jobs/{jobPublicId}/stream` remains deferred.

## API/Data Contract

```text
GET /api/v1/jobs/{jobPublicId}
GET /api/v1/evaluation-runs/{runPublicId}/events/stream
```

The hook stops polling when job status is `COMPLETED`, `FAILED`, or `CANCELLED`.
The evaluation event stream sends `event: job-event` messages whose `data` is a `JobEventResponse`.

## UI States

- Pending/running: show progress when available and status text.
- Completed: show success state and invalidate related queries.
- Failed/cancelled: show error state with retry where the caller supports retry.

## Acceptance Criteria

### AC 1 - Active job polls

```text
Platform: [WEB]
Actor: QC member

Given  a jobPublicId is available
When   useJobProgress starts
Then   it polls GET /api/v1/jobs/{jobPublicId} every 3 seconds
```

### AC 2 - Terminal job stops polling

```text
Platform: [WEB]
Actor: QC member

Given  a polled job returns COMPLETED
When   the hook handles the response
Then   polling stops
And    related TanStack Query caches are invalidated by the caller
```
