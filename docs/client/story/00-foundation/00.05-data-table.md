# Story 00.05 - Data Table

> Epic: 0 - Foundation
> Depends on: 00.04
> Backend contract source: `server/SERVER_CONTEXT.md`

## Story Header

**As a** QC member,
**I want** reusable paginated tables,
**so that** projects, datasets, results, and rubrics behave consistently.

## In Scope

- [WEB] TanStack Table wrapper.
- [WEB] Server-side pagination and sorting.
- [WEB] Loading, empty, row click, and optional selection states.

## Out of Scope

- Entity-specific column definitions.

## Deferred

- Column persistence across sessions.

## API/Data Contract

- All data tables consume `PageResponse<T>` with `items` and `totalItems`.
- Query params use Spring pageable style: `page`, `size`, `sort=field,direction`.

## UI States

- Loading rows match the column count.
- Empty state uses shared `EmptyState`.
- Mobile uses horizontal scroll only within the table container.

## Acceptance Criteria

### AC 1 - Pagination uses backend shape

```text
Platform: [WEB]
Actor: QC member

Given  a paged API returns items and totalItems
When   DataTable renders the response
Then   rows come from items
And    page controls use totalItems and totalPages
```

### AC 2 - Sorting updates API params

```text
Platform: [WEB]
Actor: QC member

Given  a sortable column header is visible
When   the user toggles sorting
Then   the next query includes sort=<field>,<direction>
```
