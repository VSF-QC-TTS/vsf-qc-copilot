# Story 00.07 - i18n Setup

> Epic: 0 - Foundation
> Depends on: 00.01
> Backend contract source: N/A

## Story Header

**As a** Vietnamese QC member,
**I want** the dashboard to default to Vietnamese with English available,
**so that** the team can work in its primary language while still supporting demos.

## In Scope

- [WEB] `next-intl` locale routing.
- [WEB] Vietnamese and English message files.
- [WEB] Language switcher.
- [WEB] Backend error code to localized message mapping.

## Out of Scope

- Backend-side translation.

## Deferred

- Additional locales beyond `vi` and `en`.

## API/Data Contract

- Backend returns stable error codes; frontend maps codes to localized messages.
- Locale routes use `/vi/...` and `/en/...`.

## UI States

- Missing message keys fail visibly during development.
- Unknown backend error code maps to a generic localized error.

## Acceptance Criteria

### AC 1 - Default locale redirects

```text
Platform: [WEB]
Actor: QC member

Given  the user opens a route without a locale prefix
When   locale middleware runs
Then   the user is redirected to the Vietnamese locale route by default
```

### AC 2 - Language switch preserves route intent

```text
Platform: [WEB]
Actor: QC member

Given  the user is viewing a dashboard route
When   they switch from Vietnamese to English
Then   the app navigates to the same route under /en
```

## Implementation Checklist

- Configure `src/i18n/config.ts` and `src/i18n/request.ts`.
- Create `messages/vi.json` and `messages/en.json`.
- Create `LanguageSwitcher`.
- Add `lib/utils/error-messages.ts`.

## Verification Notes

- Verify date formatting for Vietnamese and English.
