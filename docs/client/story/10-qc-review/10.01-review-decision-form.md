# Story 10.01 - Review Decision Form

> Epic: 10 - QC Review
> Depends on: 09.05
> Backend contract source: `server/SERVER_CONTEXT.md`

## Story Header

**As a** QC member,
**I want** to save a human review decision for an evaluation result,
**so that** final QC status reflects expert judgment rather than only AI judge output.

## In Scope

- [WEB] QC status dropdown.
- [WEB] Optional QC note.
- [WEB] Optional PIC bug user public id selector/input.
- [WEB] Upsert review decision.

## Out of Scope

- Batch review.
- User search endpoint unless backend adds one.

## Deferred

- Rich user picker for PIC bug.

## API/Data Contract

```text
PUT /api/v1/evaluation-results/{resultPublicId}/review-decision
Request: { qcStatus, qcNote?, picBugUserPublicId? }

GET /api/v1/evaluation-results/{resultPublicId}/review-decision
Response: { publicId, qcStatus, qcNote, picBug, reviewedBy, reviewedAt }
```

Writable `qcStatus` values: `PASS`, `FAIL`, `NEED_FIX`, `IGNORED`.

## UI States

- NOT_REVIEWED default when no row exists.
- Saving state.
- Success updates the table row.

## Acceptance Criteria

### AC 1 - Upsert payload uses picBugUserPublicId

```text
Platform: [WEB]
Actor: QC member

Given  the member selects a PIC bug user
When   they save the review
Then   the request payload sends picBugUserPublicId
And    it does not send picBug as the write field
```

### AC 2 - Save refreshes result row

```text
Platform: [WEB]
Actor: QC member

Given  a review decision saves successfully
When   the mutation completes
Then   the corresponding result row shows the new qcStatus
```

## Implementation Checklist

- Embed form in result detail panel.
- Map NOT_REVIEWED as display-only.
- Invalidate result list and review decision query after save.

## Verification Notes
