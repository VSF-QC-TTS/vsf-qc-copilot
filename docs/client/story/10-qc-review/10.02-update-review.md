# Story 10.02 - Update Review

> Epic: 10 - QC Review
> Depends on: 10.01
> Backend contract source: `server/API_TREE.md`

## Story Header

**As a** QC member,
**I want** to edit an existing review decision,
**so that** corrected human judgment is reflected in the result table and exports.

## In Scope

- [WEB] Read-only display of existing review.
- [WEB] Edit mode and update submit.
- [WEB] Reviewer attribution.

## Out of Scope

- Deleting review decisions.

## Deferred

- Review audit history.

## API/Data Contract

```text
PATCH /api/v1/review-decisions/{reviewDecisionPublicId}
Request: { qcStatus?, qcNote?, picBugUserPublicId? }
```

## UI States

- Existing review read-only.
- Editing existing review.
- Update loading/error/success.

## Acceptance Criteria

### AC 1 - Existing review starts read-only

```text
Platform: [WEB]
Actor: QC member

Given  a result already has a review decision
When   the panel opens
Then   the review fields are shown read-only until the member clicks edit
```

### AC 2 - Patch uses review decision id

```text
Platform: [WEB]
Actor: QC member

Given  the member edits an existing review
When   they save
Then   PATCH /api/v1/review-decisions/{reviewDecisionPublicId} is called
```
