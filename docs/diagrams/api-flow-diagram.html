Luồng xác thực (Authentication Flow)

POST Register
  1. Client POST /api/v1/auth/register với email, password, displayName
  2. Server tạo user PENDING_EMAIL_VERIFICATION
  3. Gửi email verification link: WEB_BASE_URL/verify-email?token=...
  4. Token SHA-256 hash lưu vào email_verification_tokens
POST Verify Email → Login
  1. Client POST /api/v1/auth/verify-email với raw token
  2. User status chuyển sang ACTIVE
  3. Client POST /api/v1/auth/login với email + password
  4. Response: access token (body) + refresh token (HttpOnly cookie)
POST Refresh & Logout
  1. /api/v1/auth/refresh-token: đọc cookie refresh_token, trả access token mới, rotate cookie
  2. JWT token_type=access cho protected APIs, token_type=refresh cho refresh flow
  3. /api/v1/auth/logout: xóa cookie refresh_token với Max-Age=0
POST Forgot / Reset Password
  1. /api/v1/auth/forgot-password: luôn trả 204 (không reveal account existence)
  2. Nếu email tồn tại → gửi link reset: WEB_BASE_URL/reset-password?token=...
  3. /api/v1/auth/reset-password: nhận raw token + newPassword, đặt lại mật khẩu
